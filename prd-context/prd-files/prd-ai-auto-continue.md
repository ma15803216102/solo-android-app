# PRD：AI 自动恢复对话功能

## 1. 引言/概述
在当前 Android/Capacitor App 中，AI 模型可能会因为循环判断被中断，并返回“检测到模型循环，请求已被中断。”的错误提示。为了提升用户体验，本项目旨在通过底层原生 JavaScript 注入 WebView，实时监控页面元素。当检测到该中断提示时，系统自动在输入框中填入“继续”并发送，实现对话的无缝恢复。

## 2. 目标
- 实时监控 WebView 中的 AI 回复内容。
- 准确识别“检测到模型循环，请求已被中断。”的中断标识。
- 自动向输入框填入“继续”并触发发送事件。
- 避免引起重复触发或无限死循环。

**branchName:** prd/ai-auto-continue

## 3. 用户故事

### US-001：注入监控脚本检测报错
**描述：** 作为一名 App 用户，我希望系统能在底层监控 AI 的回复内容，以便在发生特定报错时能够及时做出响应。
**依赖：** 无
**优先级：** 高

**验收标准：**
- [ ] 编写原生 JS 脚本，使用 `MutationObserver` 监听 WebView 中的 DOM 变化（如聊天记录容器）。
- [ ] 当新增的 DOM 节点文本中包含“检测到模型循环，请求已被中断。”时，触发拦截逻辑。
- [ ] 脚本具备防抖或状态锁（如 3 秒内仅触发一次），防止因 DOM 重绘导致同一条错误被重复检测。
- [ ] 在浏览器或真机 WebView 环境中验证监控逻辑生效。
- [ ] 类型检查/lint 通过

### US-002：自动输入并触发发送
**描述：** 作为一名 App 用户，我希望在检测到模型中断报错时，系统能自动在输入框输入“继续”并发送，以便无需手动点击即可恢复对话。
**依赖：** US-001
**优先级：** 高

**验收标准：**
- [ ] 脚本能够准确定位到聊天输入框（如 `textarea`、`input` 或 `[contenteditable]` 元素）。
- [ ] 自动将输入框的值设置为“继续”，并触发对应的 `input` 和 `change` 事件以适配可能存在的前端框架双向绑定。
- [ ] 准确定位到发送按钮并触发 `click` 事件；或者向输入框派发 `Enter` 键的 `KeyboardEvent`，以完成发送动作。
- [ ] 发送后自动释放状态锁，准备下一次可能的监控。
- [ ] **[仅限 UI 故事]** 在浏览器/WebView 中验证自动输入和发送（3秒内填入继续并发送，列表新增记录，监控锁释放），且不会产生死循环。
- [ ] 类型检查/lint 通过

## 4. 功能需求
- FR-1：系统必须向 WebView 注入原生 JavaScript 监听器。
- FR-2：系统必须识别特定文本：“检测到模型循环，请求已被中断。”。
- FR-3：系统必须在输入框中填入命令“继续”。
- FR-4：系统必须通过模拟点击或键盘事件触发消息发送。
- FR-5：系统必须包含节流/防抖机制，防止由于 DOM 更新引发的连续多次触发。

## 5. 非目标
- 不负责处理网络断开或超时等其他类型的报错自动重试。
- 不修改原有的后端 AI 模型逻辑，仅在客户端做自动化补救。
- 不涉及 UI 层面的样式修改或新组件开发，仅做底层脚本注入。

## 6. 设计注意事项
- 由于应用内部的页面可能使用前端框架（如 React/Vue），修改输入框值后必须派发 `Event` 才能触发数据绑定。
- 脚本注入的时机需确保在 DOM 加载完成后执行。

## 6.5 依赖说明
- 由于本项目为原生 JS 注入脚本，不引入新的第三方库。
- 依赖于原生 DOM API：`MutationObserver`，`Event` 派发。

## 7. 技术注意事项
- 技术栈：Capacitor 5 / Android WebView / 原生 Vanilla ES6+ 语法
- 性能：`MutationObserver` 需要限制观察的范围，尽量挂载在聊天列表的直接父级元素上，避免监听整个 `document.body` 带来的性能开销。
- 异常处理：需要捕获元素定位失败的情况（如找不到输入框或发送按钮），并通过 console 输出日志，避免脚本崩溃。

## 8. 成功指标
- 用户在遇到“模型循环中断”时，100% 自动恢复，无需手动干预。
- WebView 内存占用增加不超过 5MB，CPU 占用峰值增加不超过 2%（监控脚本性能开销极小）。

## 9. 开放问题
- [P0] 聊天列表容器、输入框和发送按钮的具体 DOM 选择器（class 或 id）当前尚未提供。假设开发时可通过 Inspect 确认，或采用宽泛的属性选择器进行兜底。
- [P1] 如果 AI 回复的不仅是报错文本，还混合了其他格式，是否仍触发？（当前假设：只要文本包含该错误句，就触发自动继续）。
